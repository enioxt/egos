## Documentation Standards

EGOS documentation follows these standards:

### 1. README Documentation
- **When to Update**: Upon completion of a feature, tool, or significant system component
- **Content Requirements**: High-level overview, purpose, key features, usage examples
- **Location**: Main README.md and component-specific READMEs

### 2. Roadmap Updates
- **When to Update**: During planning, milestone completion, or direction changes
- **Content Requirements**: Development stages, timelines, priorities, dependencies
- **Location**: ROADMAP.md and website/content/roadmap.md

### 3. Work Logs
- **When to Create/Update**: At task start, during development, and upon completion
- **Content Requirements**: Detailed progress, challenges, decisions, next steps
- **Location**: WORK_YYYY-MM-DD_Task_Name.md

### 4. ADRS Log
- **When to Update**: When identifying or resolving standard deviations
- **Content Requirements**: Deviation description, severity, suggested action, resolution
- **Location**: ADRS_Log.md

## How to Contribute to Documentation

1. **Follow the Standards**: Ensure all documentation follows the established standards
2. **Cross-Reference**: Include proper cross-references to related documents
3. **Update Regularly**: Keep documentation up-to-date as changes occur
4. **Maintain Consistency**: Use consistent formatting and structure
5. **Include Metadata**: Add proper metadata (title, date, author, etc.) to all documents
