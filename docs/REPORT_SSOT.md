## 2. Fundamental Principles (from br-acc SSOT v1.0.0)

1. **NUNCA ACUSAR** — Reports present public facts, never accusatory conclusions
2. **CITAR FONTE** — Every data point must have source, URL, and consultation date
3. **CONFIANCA** — Each data point has confidence: Alta (official API), Media (cross-reference), Baixa (inference)
4. **GAPS EXPLICITOS** — What was NOT found is as important as what was
5. **REPRODUTIVEL** — Anyone should be able to verify every data point following the sources

## 5. Confidence Levels

| Level | Label | Criteria | Example |
|-------|-------|----------|---------|
| `alta` | Alta Confianca | Official API, government database, verified document | Receita Federal CNPJ lookup |
| `media` | Media Confianca | Cross-referenced from 2+ sources, credible publication | News + gazette match |
| `baixa` | Baixa Confianca | Single source, inference, unverified | Social media claim |

## 6. Report Generation Rules

### Language
- Reports in Portuguese (BR) unless client requests English
- Technical terms may remain in English (e.g., "stakeholder", "compliance")
- Numbers: Brazilian format (R$ 1.000,00)
- Dates: DD/MM/YYYY

### Formatting
- Headers: `#` for title, `##` for sections, `###` for subsections
- Tables for structured data
- Code blocks for JSON/technical output
- Bold for key findings
- No emojis in formal reports (except status indicators in system reports)

### Footer (Mandatory)
```
---
Gerado por: [Model Name] via EGOS [Platform]
Data: DD/MM/YYYY HH:MM UTC-3
Report ID: REPORT-YYYY-NNN
Confianca geral: [alta|media|baixa]
Este relatorio segue o padrao REPORT_SSOT v2.0.0
```

## 9. Anti-Patterns (from br-acc REPORT_STANDARDS v1.0.0)

- **NO** reports without sources
- **NO** accusatory language or legal conclusions
- **NO** PII exposure (CPF must be masked: `***.XXX.XXX-**`)
- **NO** speculative connections without evidence
- **NO** reports generated without disclaimer
- **NO** confidence levels without justification
